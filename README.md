# Кроссворды и сканворды

Веб-приложение с генерацией русских кроссвордов и сканвордов. Фронтенд — статика,
бэкенд — Supabase (аккаунты + синхронизация прогресса). Пазлы генерируются офлайн
скриптом и складываются в БД.

## Что внутри

```
krossvordy/
├── web/                 # фронтенд (деплой как статика)
│   ├── index.html       # разметка
│   ├── styles.css       # стили
│   ├── crossword.js     # генератор кроссвордов (браузер + Node)
│   ├── scanword.js      # генератор сканвордов (браузер + Node)
│   ├── dictionary.js    # база слов с вопросами (229 шт., будет расти)
│   └── app.js           # игровая логика и UI
├── scripts/
│   ├── generate.js      # офлайн-генерация пазлов → data/puzzles.json (+ Supabase)
│   └── bench.js         # проверка корректности и замеры
├── supabase/
│   └── schema.sql       # таблицы + RLS + авто-профиль
├── docs/ARCHITECTURE.md # архитектура и дорожная карта
├── package.json
├── .env.example
└── .gitignore
```

## Требования
- **Node.js 18+** (рекомендую через [nvm](https://github.com/nvm-sh/nvm): `nvm install --lts`)
- **VS Code**. Полезные расширения: *Live Server* (запуск фронта одной кнопкой) и *ESLint*.

## Запуск в VS Code

1. Открой папку: `File → Open Folder…` → выбери `krossvordy`.
2. Открой терминал в VS Code: `` Ctrl+` `` (Терминал → New Terminal).
3. Фронтенд (играть локально), любой из вариантов:
   - **Live Server**: правый клик по `web/index.html` → *Open with Live Server*.
   - или в терминале: `npm run dev` (поднимет статику из `web/` на localhost).
4. Проверить генераторы: `npm run bench`
   Ожидаемо: ~300/300 валидных кроссвордов и сканвордов.
5. Сгенерировать пазлы в файл: `npm run gen`
   (создаст `data/puzzles.json`; число задаётся флагами
   `node scripts/generate.js --crosswords 100 --scanwords 50`).

> Сейчас фронт играет в кроссворды и сам генерит их в браузере. Сканворды уже
> генерируются кодом (`scanword.js`, см. `npm run bench`); экран-рендер сканворда —
> следующий шаг (п. 5 дорожной карты в `docs/ARCHITECTURE.md`).

## Supabase (аккаунты + синхронизация)

1. Создай бесплатный проект на [supabase.com](https://supabase.com).
2. `SQL Editor → New query` → вставь и выполни `supabase/schema.sql`.
3. `Authentication → Providers` → включи **Email**.
4. `Project Settings → API` → скопируй `Project URL` и ключи.
5. Скопируй `.env.example` в `.env` и подставь значения.
6. Залить сгенерированные пазлы в БД: `npm run gen:supabase`
   (использует `SUPABASE_SERVICE_KEY` — он секретный, только для скрипта).

Подключение фронта к Supabase (вход, чтение пазлов, сохранение прогресса) —
этап 4–5 дорожной карты; схема и заливка уже готовы.

## Деплой фронтенда (бесплатно)
- **Cloudflare Pages** или **Netlify**: подключи репозиторий, корень публикации — папка `web/`.
- Можно и **GitHub Pages** (если положить содержимое `web/` в корень/`docs`).

## Как расширять базу слов
Открой `web/dictionary.js` и добавляй строки вида:
```js
{ a: "СЛОВО", c: "подсказка-вопрос", t: "Тема" },   // буквы заглавные, без Ё
```
Чем больше база — тем плотнее и разнообразнее пазлы (особенно сканворды).
