# Архитектура

## Общая схема

```
                        ┌─────────────────────────────┐
                        │   scripts/generate.js (Node) │
                        │   генерит кроссворды/сканворды│
                        │   ОФЛАЙН (локально или CI)    │
                        └──────────────┬──────────────┘
                                       │ заливает готовые пазлы
                                       ▼
   ┌───────────────┐         ┌───────────────────────────┐
   │  web/ (статика)│  REST   │        Supabase            │
   │  Cloudflare    │◄───────►│  Postgres + Auth + RLS     │
   │  Pages/Netlify │  JS SDK │  words, puzzles, progress  │
   └───────────────┘         └───────────────────────────┘
        фронт                       бэкенд + БД
```

Тяжёлую генерацию намеренно держим **офлайн**, а не в serverless-функции: у бесплатных
функций жёсткие лимиты по времени, а заполнение плотных сканвордов прожорливо.
Сгенерированные пазлы складываются в таблицу `puzzles` и просто раздаются клиенту.

## Почему Supabase
- Нужны аккаунты + синхронизация прогресса между устройствами → встроенные Auth и
  Row Level Security дают это почти без своего кода.
- Бесплатный тариф: Postgres, авторизация, авто-REST; неактивные проекты ставятся
  на паузу, а не удаляются.

## Данные
- `words` — словарь (источник генерации): `answer, clue, theme, length, difficulty`.
- `puzzles` — готовые пазлы: `type (crossword|scanword), theme, difficulty, data (jsonb)`.
- `profiles` — профиль (создаётся триггером при регистрации).
- `progress` — введённые буквы и факт решения, по одному на (пользователь, пазл).

## Генераторы
- `web/crossword.js` — жадная упаковка слов с проверками: нет конфликтов букв,
  нет «паразитных» слов, всё связно. Штраф за габариты → компактные сетки.
- `web/scanword.js` — переиспользует движок кроссвордов и превращает гарантированно
  пустые клетки-предшественники начала слов в клетки-вопросы со стрелками (→ / ↓).
  Структура генерируется каждый раз заново (не шаблоны).

Оба работают и в браузере (как обычные `<script>`), и в Node (`require`) —
за счёт guard'а `if (typeof module !== "undefined") module.exports = ...`.

## Дорожная карта
1. ✅ Движок кроссвордов + база 229 слов + играбельный фронт.
2. ✅ Генератор сканвордов (структура + стрелки).
3. ⬜ Рост базы до 2–4 тыс. слов с вопросами (словарь/тезаурус + ИИ + проверка).
4. ⬜ Supabase: схема (готова в `supabase/schema.sql`), заливка пазлов, вход.
5. ⬜ Фронт: рендер сканворда (вопрос в клетке + стрелки), экран входа, синхронизация прогресса.
6. ⬜ Плотный backtracking-филлер для более «коммерческой» плотности сканвордов.
7. ⬜ Автогенерация по расписанию (GitHub Actions) пополняет библиотеку пазлов.
